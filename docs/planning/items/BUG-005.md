# BUG-005 — `npm run test -- --coverage` revienta al parsear `globals.css` y las guías `.md`

## Descripción

El comando de cobertura —el que corre CI y el que pide el protocolo de Bolt antes de entregar a Scooby— termina en error:

```
Failed to parse file:///F:/Mios/adoptia/src/app/globals.css. Excluding it from coverage.
 Error [RolldownError]: Parse failed with 1 error:
    at V8CoverageProvider.remapCoverage (@vitest/coverage-v8/dist/provider.js:133:10)
    at V8CoverageProvider.getCoverageMapForUncoveredFiles (…:109:4)
```

Lo mismo con los `.md` de `src/content/guias/`. El proveedor de cobertura V8 intenta incluir en el informe ficheros que no son JS/TS (`globals.css`, guías Markdown) al calcular los "uncovered files", y rolldown no puede parsearlos.

`npm run test` (sin `--coverage`) pasa sin problema: **759 tests en verde**. Solo falla el informe de cobertura.

## Contexto / impacto

Detectado durante FEATURE-022 y **confirmado como preexistente**: se reprodujo con `git stash` sobre el árbol limpio, sin ningún cambio del item. No lo causó FEATURE-022.

Impacto: no se puede medir la cobertura ni comprobar los umbrales (70% global, 80% en `src/lib/`) que `vitest.config.ts` define y que CI debería hacer cumplir. Es decir, el umbral de cobertura del proyecto lleva un tiempo sin vigilar de verdad — conviene mirar también si CI está enmascarando el fallo.

Pista de arreglo probable: `coverage.include`/`exclude` en `vitest.config.ts` restringido a `src/**/*.{ts,tsx}`, o `coverage.allowExternal`/`extension`. Verificar antes si CI está en rojo o si lo traga.
