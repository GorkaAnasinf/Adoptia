# BUG-005 — `npm run test -- --coverage` revienta al parsear `globals.css` y las guías `.md`

## Descripción

El comando de cobertura —el que corre CI y el que pide el protocolo de Bolt antes de entregar a Scooby— termina en error:

```
Failed to parse file:///F:/Mios/adoptia/src/app/globals.css. Excluding it from coverage.
 Error [RolldownError]: Parse failed with 1 error:
    at V8CoverageProvider.remapCoverage (@vitest/coverage-v8/dist/provider.js:133:10)
    at V8CoverageProvider.getCoverageMapForUncoveredFiles (…:109:4)
```

Lo mismo con los `.md` de `src/content/guias/`. El proveedor de cobertura V8 intenta incluir en el informe ficheros que no son JS/TS (`globals.css`, guías Markdown) al calcular los "uncovered files", y rolldown no puede parsearlos.

`npm run test` (sin `--coverage`) pasa sin problema: **759 tests en verde**. Solo falla el informe de cobertura.

## Contexto / impacto

Detectado durante FEATURE-022 y **confirmado como preexistente**: se reprodujo con `git stash` sobre el árbol limpio, sin ningún cambio del item. No lo causó FEATURE-022.

Impacto: en local no se puede medir la cobertura ni comprobar los umbrales (70% global, 80% en `src/lib/`), que es un paso obligatorio del protocolo de Bolt antes de entregar a Scooby.

**Corrección (2026-07-15, tras revisar los logs de CI)**: al abrir este item se supuso que CI también estaba afectado y que «el umbral llevaba tiempo sin vigilarse». **Es falso.** El `RolldownError` aparece igual en CI (`/home/runner/…/globals.css`), pero en Linux es solo ruido: el proceso sale 0, la tabla de cobertura se imprime y los umbrales se evalúan (el run `29410086226`, anterior a este arreglo, muestra `All files` y ningún `ERROR: Coverage`). **El fallo era exclusivo de Windows**, donde el mismo error tumba el proceso con exit 255. La cobertura de CI nunca dejó de vigilarse.

Pista de arreglo probable: `coverage.include`/`exclude` en `vitest.config.ts` restringido a `src/**/*.{ts,tsx}`, o `coverage.allowExternal`/`extension`. Verificar antes si CI está en rojo o si lo traga.

## Cierre (2026-07-15)

- **Causa**: `coverage.include` era `["src/**"]` — todo, no solo código. Para calcular los "uncovered files" (los que ningún test importa), el proveedor v8 parsea cada fichero incluido como JS; `globals.css` y las guías de `src/content/guias/*.md` no lo son y rolldown aborta **el informe entero**, no solo ese fichero. Los mensajes `Excluding it from coverage` despistaban: parecían avisos benignos, pero el `RolldownError` posterior tumbaba el proceso con exit 255.
- **Arreglo**: `include: ["src/**/*.{ts,tsx}"]` en `vitest.config.ts`. Un fichero, cuatro líneas. La lista de `exclude` se deja intacta.
- **Hallazgo tranquilizador**: no había deuda de cobertura escondida detrás del error. Con el informe ya generándose, los umbrales se cumplen **sin tocar ni un test**: 80,85% statements · 75,69% branches · 74,22% functions · 82,44% lines (umbral global 70%), y `src/lib` al 96,6% (umbral 80%). El error impedía *ver* la cobertura, no la había degradado.
- **Verificación**: `npm run test -- --coverage` pasa de exit 255 a **exit 0**, con 759 tests verdes. Lint y `tsc --noEmit` limpios.
