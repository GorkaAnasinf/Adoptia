# BUG-006 — El listado sirve la URL de YouTube como imagen de portada

## Descripción

En el listado de animales, la imagen de un animal que tenga vídeo de YouTube pero **ninguna foto marcada como portada** es la URL del vídeo (`https://youtu.be/…`) metida en el `src` de la imagen. Resultado: imagen rota en la tarjeta del animal.

FEATURE-020 había resuelto esto filtrando explícitamente por fotos en el RPC `animals_search`:

```sql
select m.url from public.animal_media m
where m.animal_id = a.id
  and m.type = 'photo'          -- <-- esta línea
order by m.is_cover desc, m.sort_order asc
limit 1
```

**IMPROVEMENT-021 reescribió la función para añadir `p_query` y perdió el filtro `and m.type = 'photo'`** (migración `20260714150000_improvement021_animals_search_query.sql`, subconsulta de `cover_url`). Ninguna otra parte cambió.

Reproducido en local contra el stack real (2026-07-15):

```sql
-- animal con un youtube y sin foto de portada
select id, name, cover_url from public.animals_search(p_limit := 50) where id = '…3301';
--  id  | name |     cover_url
-- …3301| Luna | https://youtu.be/x     <-- debería ser null o una foto
```

## Contexto / impacto

- **Está en producción desde el 2026-07-14** (IMPROVEMENT-021 se desplegó ese día).
- Afecta a la primera impresión del catálogo: la tarjeta del animal es lo que ve el adoptante antes que nada.
- Alcance real: solo animales con vídeo y sin foto de portada. Con datos de demo puede pasar desapercibido; con datos reales de protectora, no.

## Segundo hallazgo — el test que debía cazarlo está roto

`src/test/rls/animal-media.test.ts` → *"animals_search devuelve una foto como cover_url aunque el vídeo tenga sort_order menor"* falla, pero **por un motivo distinto al bug**: su `insert` masivo pasa objetos con claves distintas (uno sin `is_cover`, otro con él) y **PostgREST exige que todas las filas de un insert masivo tengan las mismas claves**. La inserción falla entera, el test no comprueba `error`, se queda sin medios y `cover_url` sale `null`.

Es decir, el test **nunca ha ejercitado el escenario que describe**. Y no saltó ninguna alarma porque los tests de RLS se saltan solos sin las variables `SUPABASE_TEST_*` (Decisión #17) — conviene comprobar si CI las tiene configuradas o si toda la suite de RLS lleva tiempo sin ejecutarse ahí.

## Plan de desarrollo

### Modelo de datos

- Migración `20260715160000_bug006_cover_url_solo_fotos.sql`: `create or replace` de `animals_search` restaurando `and m.type = 'photo'` en la subconsulta de `cover_url`. La firma no cambia respecto a IMPROVEMENT-021, así que no hay overload que soltar (a diferencia de aquella, que sí tuvo que hacer `drop`).

### Tareas TDD

1. Arreglar el test roto para que ejercite el escenario: mismas claves en todas las filas del insert masivo y `expect(error).toBeNull()`.
2. Test nuevo del caso que fallaba de verdad: vídeo sin foto → `cover_url` null.
3. Ver el rojo (`expected 'https://youtu.be/…' to be null`), luego la migración.
4. Comprobar que el test muerde: quitar el filtro a mano en BD y verificar que falla.

### Dependencias

- Ninguna.

## Criterios de aceptación / Casuística a cubrir

- [x] Un animal con vídeo y sin foto de portada devuelve `cover_url` nulo (la tarjeta cae al placeholder), nunca la URL del vídeo.
- [x] Un animal con vídeo (`sort_order` menor) y foto de portada devuelve la foto — con el insert del test arreglado, o sea ejercitando el escenario de verdad.
- [x] El test falla si se vuelve a quitar el filtro `type = 'photo'` (comprobado quitándolo a mano en BD: falla con `expected 'https://youtu.be/dQw4w9WgXcQ' to be null`).
- [x] Los inserts de fixtures del test comprueban `error`.

## Cierre (2026-07-15)

- **Arreglo**: migración `20260715160000_bug006_cover_url_solo_fotos` restaura `and m.type = 'photo'`. Sin foto, `cover_url` es null y la tarjeta usa el placeholder.
- **El TDD salió limpio**: al arreglar el insert del test viejo, este pasó a verde solo — prueba de que su fallo era el `insert` y de que nunca había ejercitado nada. El test nuevo falló con el síntoma real del bug, y la migración lo puso en verde.
- **Suite: 882 verdes, 150 ficheros, ninguno saltado** — con el stack local levantado, por primera vez en este ciclo la suite entera (RLS incluida) está limpia. `tsc` y `lint` limpios.
- **Revisión del punto 3 de las notas** (más inserts masivos con claves desiguales): no se encontró ninguno más; el resto de inserts de fixtures son de fila única.
- **Follow-up que no cierra este item**: los tests de RLS se saltan solos sin `SUPABASE_TEST_*` y Vitest no carga `.env`, así que basta con no exportarlas para que 122 tests desaparezcan sin ruido. Este bug vivió en producción justo por eso. Queda anotado en el BACKLOG: comprobar si CI los ejecuta de verdad.
