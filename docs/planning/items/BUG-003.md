# BUG-003 — El mapa del alta se ve en gris

## Descripción

En el paso de ubicación del alta de protectora, el mapa Leaflet se muestra **en gris** (sin tiles). Causa: la **Content-Security-Policy** de `next.config.ts` tiene `img-src 'self' data: blob: https://*.supabase.co`, que **no permite** cargar las tiles de OpenStreetMap (`https://{a,b,c}.tile.openstreetmap.org`, servidas como `<img>`). Los iconos de marcador (hoy desde `unpkg.com`) también quedan bloqueados.

## Contexto / impacto

Rompe el paso 2 del onboarding en producción: la protectora no puede ver ni ajustar el pin de su ubicación. Bloquea el geocoding manual.

## Plan de desarrollo

### Fix

1. **CSP**: añadir `https://*.tile.openstreetmap.org` a `img-src` en la CSP. (Quita el gris.)
2. **Iconos de marcador locales**: copiar `marker-icon.png`, `marker-icon-2x.png`, `marker-shadow.png` de `leaflet/dist/images` a `public/leaflet/` y apuntar `L.icon` ahí, en vez de `unpkg.com` (evita otra excepción CSP; coherente con "coste 0 / sin CDNs").
3. **`invalidateSize()`**: llamar tras montar el mapa por si el contenedor arranca con tamaño 0 (dynamic import / step recién montado).

### Refactor testeable

- Extraer la CSP/headers a `src/lib/security-headers.ts` (importado por `next.config.ts`) para poder testear que `img-src` incluye el host de tiles y que `connect-src` sigue permitiendo Supabase.

### Tareas TDD

1. Test de `security-headers`: `img-src` incluye `*.tile.openstreetmap.org`; se mantienen las directivas de Supabase (`connect-src`) y Turnstile (`frame-src`/`script-src`).
2. Iconos locales en `public/leaflet/` + `MapPinPickerInner` apuntando a ellos.
3. `invalidateSize` tras montar.
4. **verify**: arrancar la app y comprobar que el mapa renderiza tiles reales y el pin es arrastrable.

### Dependencias

- Ninguna. Independiente de [[FEATURE-018]] / [[IMPROVEMENT-002]] (el rediseño del mapa vive allí; esto solo lo desbloquea).
